# Sprut Meeting Recorder v2 — приложение открывает ваш реальный сайт

Домен: https://ononeon.com/meeting.php (HTTPS уже работает — отлично,
это упрощает конфигурацию, никаких доп. костылей с cleartext не нужно).

## Главное отличие от первой версии

Приложение не возит копию HTML внутри себя. Оно открывает вашу настоящую
страницу `https://ononeon.com/meeting.php` прямо с сервера (как встроенный
браузер) — весь ваш PHP-бэкенд, чек-листы, авторизация, api.php работают
как есть, без дублирования кода.

Нативный Android-модуль (foreground service) добавляет только одну вещь:
запись звука, которая не прерывается при блокировке экрана.

## ДВЕ РАЗНЫЕ вещи нужно загрузить в ДВА РАЗНЫХ места

### 1. На ваш сервер — файл из папки `server-side-file-to-upload/`

Замените `meeting.php` на сервере (`/var/www/call-audit-app/public/meeting.php`,
судя по вашему DEPLOY-README) на файл `server-side-file-to-upload/meeting.php`
из этого архива — через FTP/SCP/SFTP, как обычно деплоите.

Что изменилось: та же страница, что была, плюс мост к нативному
Android-плагину (работает только внутри приложения; в обычном браузере
всё как раньше, без изменений).

**Этот файл НЕ должен попадать в GitHub** — в нём захардкожен пароль входа.

### 2. В GitHub-репозиторий — всё остальное содержимое архива

`www/`, `scripts/`, `.github/`, `package-android-src/`, `package.json`,
`capacitor.config.json`, `.gitignore`, `README.md`
(всё, КРОМЕ папки `server-side-file-to-upload/`).

Дальше как раньше: Actions подхватит workflow сам → дождаться зелёной
галочки → скачать APK из Artifacts.

## Проверка на устройстве

1. Установите APK, войдите под тем же логином/паролем, что и на сайте
2. Нажмите «Начать встречу», заблокируйте экран на 20-30 сек
3. Разблокируйте — уведомление «Идёт запись встречи» должно было висеть
   всё это время
4. «Закончить встречу» — должно уйти на анализ как обычно

Если Samsung прервал запись — Настройки -> Батарея -> Ограничения фона ->
приложение -> «Не ограничивать».

## Что отключено в этой версии

Live-прослушивание руководителем (WebRTC) — работает только фоновая
запись + отправка на анализ после завершения встречи.
